# Docker Compose installation

Docker deployment is supported on Linux with host networking. Host networking is used because Bedrock uses UDP listeners and Full Proxy routes may expose several independent ports.

## Build and start

```bash
cd deploy/docker
cp .env.example .env
```

Set long random values for the default and per-backend bridge secrets. `SESSION_CORE_TOKEN` may be left empty and generated by the runtime.

Start:

```bash
docker compose up -d --build
docker compose logs -f ninjos-proxie
```

The image is based on Node.js 22 and includes:

- Ninj-OS Edge Shield
- Dashboard
- Full Proxy Session Core and pinned npm dependencies
- Vanilla Bridge `.mcpack`
- Linux and Windows vanilla host agents
- Documentation

## Persistent data

Named volumes preserve:

```text
/opt/ninjos/config
/opt/ninjos/runtime
```

Do not run `docker compose down -v` unless those volumes should be destroyed.

## First login

Read the setup code from container logs or:

```bash
docker compose exec ninjos-proxie cat /opt/ninjos/runtime/FIRST_RUN_SETUP.txt
```

Open `http://HOST-IP:25571`.

## Ports

The compose file uses `network_mode: host`, so Docker does not publish individual ports. Open the required UDP listeners and dashboard TCP port in the host firewall. Private offline-mode backend ports must remain unexposed.

## Upgrade

```bash
git pull
docker compose build --no-cache
docker compose up -d
```

The named volumes remain attached.
